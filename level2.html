Password Cracking Basics

Before we can touch specifically on password cracking, we must understand what hashing is.

What is hashing?

Passwords and other sensitive data are not stored in plaintext, meaning if your password is abc123, a database will not store that password as abc123.

Instead, the plaintext is converted into a fixed-size string of characters using cryptographic algorithms. The idea is that the hash value is stored, and you cannot easily retrieve the original plaintext password from the hash.

There are a lot of hashing algorithms out there. Here are a few popular hash types that you see in CTFS:

  • MD5, SHA (SHA256, SHA2, etc.), CRC

Why do we use hashing?

Hashing is used to securely store passwords in databases. If a hacker were to get access to a company's database and every sensitive piece of data was stored in plaintext, this would increase risk of data breaches that are hard to recover from.

Think of it like this: You wouldn't want a hacker to easily access something like your password, right? That's why hashing provides an extra layer of security.

A Little Information about MD5

MD5 is a type of hash function algorithm, generating a 128-bit hash value (represented as a 32 character string) from any input of data.

  • Plaintext: password123
  • MD5 Hash: 482c811da5d5b4bc6d497ffa98491e38

Cracking an MD5 Hash

What is Password Cracking?

Password Cracking is the process of trying to find the original plaintext from a hash, often using techniques like brute-forcing or predefined wordlists.

How Attackers User MD5 Hashes:

  1. Brute-Force: Trying every possible combination of characters until the hash matches.
  2. Dictionary Attacks: Use a list of common passwords (a.k.a a dictionary) to see if any match the hash.

BE WARNED! MD5 is considered insecure due to how fast hackers can crack the hashes using methods like brute-force attacks. This makes MD5 unsuitable for modern security standards.
